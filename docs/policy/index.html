PlanLang Policy Examples

Authorize typed plans with declarative policies in CEL or Rego. Reference policies by policy:// URIs inside PlanLang documents.

CEL — Low-risk email

allow {
  input.capability.tool == "email"
  input.constraints.riskLevel == "low"
  input.constraints.budget.max <= 200
  input.steps.all(step, step.effects.all(effect, effect in ["email.sent"]))
}

Rego — CRM mutation

package praeon.crm

default allow = false

allow {
  input.capability.tool == "salesforce"
  input.capability.name == "update"
  input.constraints.budget.max <= 100
  input.steps[_].preconditions[_] == "crm.allowlist"
}

Least-privilege scopes

  • Scope on domains (domain:example.com).
  • Limit records (record:claim-*).
  • Require approvals for escalations.

Policy lifecycle

  • Policies checked into version control.
  • Policy digests captured in operational traces.
  • Drift alerts when PlanLang references stale digests.
