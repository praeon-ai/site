Bento / BentoCloud Integration

Init-container verification for Kubernetes; CI step for Bento builds. Loader-native plug-in behind a feature flag.

Mode A — Kubernetes (initContainer gate)

Verify at pod start. If policy denies or CRL is stale, the app container never starts.

apiVersion: apps/v1
kind: Deployment
metadata:
  name: bento-svc
spec:
  replicas: 1
  selector: { matchLabels: { app: bento-svc } }
  template:
    metadata: { labels: { app: bento-svc } }
    spec:
      initContainers:
      - name: praegate
        image: ghcr.io/praeon/gate:preview
        args:
          - gate
          - verify
          - --manifest=/man/model.mwal.json
          - --policy=/pol/prod.hcl
          - --runtime=bento
          - --load=/w/llama3.1-q4k.gguf
          - --out-store=/verified
        volumeMounts:
          - { name: manifests, mountPath: /man, readOnly: true }
          - { name: policies,  mountPath: /pol, readOnly: true }
          - { name: weights,   mountPath: /w,   readOnly: true }
          - { name: verified,  mountPath: /verified }
      containers:
      - name: app
        image: your-bento-image:tag
        env:
          - name: PRAEON_MODELS
            value: /models
        volumeMounts:
          - { name: verified, mountPath: /models, readOnly: true }
      volumes:
        - { name: manifests, configMap: { name: praeon-manifests } }
        - { name: policies,  configMap: { name: praeon-policies  } }
        - { name: weights,   persistentVolumeClaim: { claimName: weights-pvc } }
        - { name: verified,  emptyDir: {} }

App reads only from the verified mount. No verified weights → no start.

Mode B — CI step for Bento build

Add a verification step before baking the Bento. If it fails, the build fails.

# .github/workflows/bento.yml (excerpt)
- name: Verify model with Praeon
  run: |
    praeon gate verify \
      --manifest ./manifests/model.mwal.json \
      --policy   ./policies/prod.hcl \
      --runtime  bento \
      --load     ./weights/llama3.1-q4_k_m.gguf \
      --out-store ./verified
- name: Build Bento
  run: |
    bentoml build
    # ensure Bento reads from ./verified at runtime

Mode C — Local preview

# Verify + stage to ./verified
praeon gate verify \
  --manifest ./model.mwal.json \
  --policy   ./policy.hcl \
  --runtime  bento \
  --load     ./weights/llama3.1-q4_k_m.gguf \
  --out-store ./verified

# Run service pointing to verified store
PRAEON_MODELS=./verified bentoml serve service:svc

Receipts, CRL, and Fail-closed

  • Every admit emits a signed receipt (JWS + timestamp). Export to Splunk/ELK/Datadog.
  • Gate enforces current crl_version; on fetch failure, fail-closed (optional maintenance mode window).
  • Optional hardware attest (Intel TDX/Azure CC) per policy; disabled by default.
# Export last receipt
praeon receipt export --to elk --file ./receipts/last.json

Operational tips

  • Pin manifest/profile versions in ConfigMaps for reproducibility.
  • Scope policies per env (dev/stage/prod) and approver quorum (e.g., 2-of-3).
  • Set resource requests for the initContainer to avoid scheduling delays.
