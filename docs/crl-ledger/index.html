Trace Ledger & Revocation

Praeon maintains canonical trace hashes and adapter digests so revocation and replay remain deterministic.

Ledger anchors

  • Trace hash stored in append-only ledger.
  • Periodic anchors to customer-controlled transparency logs.
  • Adapter digest + policy digest recorded per plan.

Revocation workflow

  1. Flag trace ID for revocation (policy breach or credential compromise).
  2. Publish revocation entry with reason codes.
  3. Notify adapters and orchestrators via webhook + SIEM event.

Target SLOs

  • Revocation propagation < 60 seconds fleet-wide.
  • Ledger anchor issuance every 15 minutes.
  • Replay validation available within 5 minutes of revocation.
