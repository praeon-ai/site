CRL & Transparency Log

Revocation and proof of inclusion for receipts. Target propagation < 60 s across fleet.

Model

  • Pub/sub fanout with regional caches; signatures verified at edge.
  • Monotonic crl_version; clients refuse stale versions by policy.
  • Receipts anchored in an append-only log; periodic hash anchors.

Failure handling

  • On CRL fetch failure: fail-closed (configurable maintenance mode).
  • On log unavailability: admit only previously-valid artifacts within TTL window.
