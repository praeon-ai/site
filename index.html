What is MWAL-1?

  • Signed manifest (C2PA/CycloneDX/SPDX profile)
  • Verify before load with atomic FD semantics
  • Policy gates (who/what/where) at load-time
  • Quorum-signed manifests · rapid CRL
  • Audit receipts: pseudonymous, TTL retention

Targets: 1 GB model P95 ≤ 200 ms decision, CRL propagation < 60 s.

How it works

dev / ops
   │   push signed manifest
   ▼
Praeon Gate ── verify ▶ policy ▶ admit/deny
   │
   ├─ if admitted ▶ atomic load of weights
   │
   └─ emit receipt ▶ Praeon Ledger ▶ auditors / SIEM

Admits only attested artifacts. Everything else is denied by default.

Quick start

# 1) Install Gate (example CLI)
curl -L https://get.praeon.ai/gate.sh | bash

# 2) Verify & load with MWAL-1
praeon gate verify \
  --manifest ./model.mwal.json \
  --policy ./policy.hcl \
  --runtime vllm \
  --load ./weights.bin

# 3) Receipt (JSON)
cat ./receipts/2025-10-31T12-07Z.json

Sample commands shown. We’ll tailor integration to your runtime (vLLM/Ollama/Bento).
